Повторяющиеся вычисления

  • Ранее мы рассматривали только линейные вычисления
  • Код исполнялся от начала до конца
  • Даже вызовы функций не меняли этого факта
  • Внутри функций все тоже было линейным
  • Но этого явно мало для полезных и осмыленных действий
  • Нужно уметь "возвращаться назад"

Циклы

  • Python предлагает несколько вариантов циклических вычислений
  • Простейший по форме - цикл while
  • Синтаксически почти неотличим от if
  • Только другое ключевое слово
  • Семантически - отличие по сути в одном
  • После выполнения блока снова идем на проверку условия
  • И если оно истинно - снова исполняем блок
  • И так - пока условие не станет ложным

Пример



while False:
    print('never')

while True:
    print('forever')

Пример



s = '' 

while s != 'exit':
    s = input()
    if s != 'exit':
        v = int(s)
        print(v * v)

Проблема

  • Проблема в последнем примере - повторение одного условия
  • Мы в середине итерации поняли, что надо завершать цикл
  • Но до проверки условия нам нужно дойти до конца итерации
  • И приходится применять одно условие дважды
  • Любое повторение логики ухудшает качество кода

Вариант 1

  • Совместить чтение строки с проверкой условия
  • Но нам же нужно сохранить результат input()
  • И одновременно использовать его в выражении
  • Знакомая нам форма присваивания не поможет
  • Потому что в ней присваивание не является выражением
  • Классическое присваивание - это statement, с выражением справа от знака =

Вариант 1

  • Еще есть присваивание как выражение
  • Обозначается как :=
  • Значение правой части становится значением присваивания

b = (a := 1) + 1
print(a, b)

Пример


        
s = ''  
        
while (s := input()) != 'exit':
    v = int(s)
    print(v * v)

Вариант 2

  • Вариант 1 как минимум не всегда подходит объективно
  • И он по духу не совсем питоновский
  • Объективно не подходит - когда причина раннего завершения цикла отличается от того, что является обычным условием продолжения работы
  • Пример: интеракивное исполнение неких команд от пользователя
  • exit - это команда на завершение и ее можно проверять в while
  • Но при исполнении другой команды мы можем обнаружить, что обрабатываемые данные некорректны
  • И завершить работу

Вариант 2

  • break - предложение, обозначающее выход из циклв
  • Им не стоит злоупотреблять
  • Если можно обойтись без него - лучше обойтись
  • И нормальное использование break - в рамках условного предложения /li>
  • Когда по логике нам нужно завершить цикл посреди итерации

Пример



while True:
    s = input()
    if s == 'exit':
        break
    v = int(s) 
    print(v * v)

Объяснение

  • В примере break используется уместно
  • Потому что по смыслу итерация состоит из чтения данных и исполнения
  • И хотя бы одно чтение всегда нужно
  • Поэтому while True оправдано
  • И решение о выходе принимаем между чтением и исполнением - по смыслу происходящего

Пример



def gcd(a, b):
    """
    Наибольший общий делитель
    Алгоритм Евклида
    a, b - неотрицательные целые
    """
    while a: # используем логический контекст
        a, b = b % a, a # групповое присванивание
    return b

Пример



def gcd(a, b):
    """
    Наибольший общий делитель
    Алгоритм Евклида
    a, b - неотрицательные целые
    """
    while True:
        if not a:
            break  # плохое использование break
        a, b = b % a, a
    return b

continue

  • Еще одно вспомогательное предложение - continue
  • Тоже имеет смысл только по какому-то условию
  • Завершает текущую итерацию и идет снова на проверку условия
  • Например, в интерактивном приложении читаем вводимые команды
  • Если получили exit - то делаем break
  • Если получили пустую строку - делаем continue
  • Если команда знакомая то исполняем

Пример



while True:
    command = input()
    if not cmd:
        continue
    if cmd == 'exit':
        break
    ... # обрабатываем команду

Составные типы данных

  • В реальных задач редко дело ограничивается одиночными данными
  • Редко дело ограничивается вычисление одной формулы с числами
  • Чаще нам нужно обрабатывать наборы данных
  • Например данные о некоторых событиях: покупках, результатах контрольных и т.п.
  • Нужно уметь пройтись по всем событиям
  • Появляется потребность в структурах данных для хранения наборов объектов

Составные типы данных

  • И эта потребность многослойна
  • На первом уровне - наборы событий/фактов
  • На каждый отдельный факт или событие - тоже не сводится к одному объекту простого типа
  • У факта или события есть атрибуты и их значение
  • Это может быть дата, имена участников, количество купленных товаров (если это покупка)
  • Каждый элемент набора содержит в себе тоже какие-то элементы

Составные типы данных

  • Дата события - уже ближе к элементарным типам данных
  • В простом случае она состоит из дня, номера месяца и года
  • Но может хранить или не хранить время
  • С разной точностью
  • С привязкой к часовому поясу или без таковой
  • С привязкой к календарной системе или без таковой
  • В общем случае все может быть непросто

Составные типы данных

  • На каком-то уровне разбиения на элементы приходим к строковым элементам
  • Со строками мы уже встречались - в силу их базовости
  • И еще встретимся подробнее
  • Но строка - тоже составной тип
  • Набор символов

Составные типы данных

  • Ключевые составные типы данных будут разобраны в деталях
  • И научимся строить свои
  • Многие типы имеют общие свойства
  • Например, по объектам многих составных типов можно пройтись циклом
  • И это отдельное составное предложение со своим стартовым ключевым словом - for

Пример



for v in range(10):
    print(v, v * v)

for c in 'abcdefghiji1234!@#':
    print(c, ord(c))

# эту конструкцию пока не разбираем
with open('data.txt') as f:
    for s in f:
        print(len(s))

Разберем

  • Все циклы for в примере структурно похожи
  • Везде есть ключевые слова for и in
  • Везде есть двоеточие в конце первой логической строки
  • Везде есть имя переменной между for и in
  • Везде есть что-то между in и двоеточием
  • Везде есть блок со смещением
  • И в блоке упоминается переменная, упомянутая в первой строке

Разберем

  • Справа от in - выражение (в общем случае)
  • Результатом выражения должен быть объект итерируемого типа
  • Примеры НЕитерируемых типов: int, float, bool
  • Итерируемые типы из примера выше: range(диапазон), str(строка), file object(файловый объект)
  • Другие примеры итерируемых типов: список, кортеж, множество, словарь, генератор

Разберем

  • Слева от in в примерах - имя переменной
  • В общем случае там может быть более сложная конструкция
  • В ходе цикла for перебираются значения итерируемого типа
  • На каждое значение исполняется блок кода
  • Перед очередным исполнением блока обрабатываемое значение сохраняется в переменную
  • Имя переменной - то, что указано сразу после for

range

  • range представляет диапазон целых чисел
  • Есть три формы задания range: с одним, двумя или тремя параметрами
  • Простейшая форма - с одним параметром: range(n)
  • Если n не положительно - это пустой диапазон
  • Если n положительно - диапазон включает 0, 1, ..., n - 1
  • n в диапазон не входит 

range

  • С двумя параметрами можно задавать левую и правую границы отрезка
  • Левая граница включается, правая - нет
  • Не запрещено правую границу делать меньше или равной левой
  • Получится пустой диапазон

        
for v in range(5, 10): 
    print(v, v * v)

range

  • Если есть третий параметр - это шаг
  • По умолчанию шаг равен 1
  • Шаг 0 приводит к исключению
  • При положительном шаге k в диапазон в ходит каждое k-е число, начиная c левой границы
  • Пока число меньше правой границы

for v in range(5, 14, 3):
    print(v, v * v)
# 5 25
# 8 64
# 11 121

range

  • Шаг может быть отрицательным
  • При отрицательном шагу у непустого диапазона правая граница должна быть меньше левой
  • Левая граница всегда попадает в непустой диапазон
  • Дальше проверяются все числа с отрицательным шагом k
  • Пока число больше правой границы

for v in range(14, 5, -3):
    print(v, v * v)
# 14 196
# 11 121
# 8 64

Строка

  • Строка - последовательность символов
  • Цикл for по строке - перебор символов


for c in 'abcdefg':
    print(c, ord(c))

Файловый объект

  • Тоже итерируемый объект
  • Со своими особенностями
  • Создается путем открытия файла на чтение
  • В нашем примере это делает функция open
  • Количество файлов, которые может держать открытыми одна программа, ограничено
  • Файлы, которые больше не нужны, надо закрывать
  • with в примере - менеджер контекстов
  • При его использовании просходит закрытие файла по завершении блока
