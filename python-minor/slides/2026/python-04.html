Строки

  • Один из ключевых типов данных
  • Можно задавать непосредственно в коде строковые литералы
  • Можно порождать новые строки с помощью операций
  • Можно читать строки из файлов
  • Можно записывать строки в файлы

Пример литералов


s1 = 'abc'
s2 = "abc"
s3 = "O'Henry"
s4 = "abc\ndef"
s5 = "abc\
def"
s6 = "abc\\def"
s7 = "abc" "def"
s8 = "abc" \
     "def"
s9 = ( "abc"
       "def" )

Базовые правила

  • Начало и конец ограничиваются одинарными или двойными кавычками
  • Начальные и конечные кавычки должны быть на одной физической строке - с одним нюансом
  • Можно в незакрытом строковом литерале использовать \ перевод новой строкой
  • Тогда содержимое строкового литерала продолжается на следующей строке
  • И завершающая кавычка должна быть на следующей строке
  • Это похоже на продолжение логической строки, но есть отличия в семантике

Базовые правила

  • Внутри кавычек \ имеет особое значение
  • \ с последующим символом обозначает что-либо
  • То, для чего нет видимого символьного представления
  • То, что при обычном представлении вызывает неоднозначность
  • \n - перевод строки
  • \t - табуляция
  • \', \" - одинарная или двойная кавычка как часть строки
  • \\ - \ как символ

Базовые правила

  • Два строковых литерала подряд склеиваются вместе
  • Вариант применения - длинная строковая константа
  • Можно представить на нескольких физических строках
  • Но нужно для этого организовать логическую строку
  • Через скобки или \

Длинные строковые константы

  • Ограничиваются тремя подряд одинарными или двойными кавычками
  • Растягиваются на много физических строк
  • Перевод строки внутри такого литерала превращается в перевод строки внутри значения строки
  • Не нужно использовать \ для отдельно стоящей кавычки внутри такого литерала

Пример



s1 = """
Однажды в студеную зимнюю пору
Я из лесу вышел
Был сильный мороз
"""

s2 = """
"To be, or not to be, that is the question" 
Shakespeare, William. Hamlet
"""

Буквенные префиксы

  • Перед кавычками можно использовать букву f или r
  • Поотдельности или вместе
  • Перед любыми - двойными или одинарными, простыми или тройные
  • r отключает интерпретацию \ как специального символа
  • f включает режим интерполяции: PEP 498 – Literal String Interpolation
  • В режиме интерполяции можно внутрь строки вкраплять вычисляемые выражения

Пример



s1 = r"\"
s2 = r"\n"

for v in range(10, 20):
    print(f'{v}^2 is {v * v}')

s3 = fr's1 contains \: {s1}'

Взятие символа но номеру

  • Можем взять значение символа по номеру
  • Например: s[0], s[3], s[i]
  • Номер символа принято называть индексом
  • Индекс должен быть целым числом
  • Начальный символ строки имеет индекс 0
  • Длина строки определяется функцией len
  • len(s) - 1 - максимальный корректный индекс для непустой строки
  • Это индекс последнего символа строки
  • (У пустой строки корректных индексов нет)

Пример



s = "abcdefghij"

print(s[0])
print(s[1])
print(s[8])
print(s[9])
print(s[len(s) - 1])

Отрицательный индекс

  • Довольно часто требуется взять элемент в конце строки
  • И/или с нумерацией позиции относительно конца
  • Последний, предпоследний, третий с конца, ...
  • Для такого случая появилась схема отрицательных индексов
  • Последний символ можно обозначить как элемент с индексом -1
  • Предпоследний: -2
  • Третий с конца: -3

Пример


    
s = "abcdefghij"

print(s[-1])
print(s[-2])
print(s[-9])
print(s[-10])

Отрицательный индекс

  • Отрицательный индекс часто бывает удобным
  • Но этот механизм коварен
  • Допустим, что индекс вычисляется
  • И в логике вычисления индекса есть ошибка
  • В результате ошибки получаем индекс на 1 меньше требуемого
  • Если нужно обратиться по индексу 0 и случайно попадаем в индекс -1 - можем не заметить ошибки

Отрицательный индекс

  • Если индекс i корректен, то кажется что индекс -i тоже корректен и указывает на "симмемтричную" позицию
  • Но есть нюансы
  • Если i == 0, то -i указывает на ту же позицию
  • Если i == -len(s), то -i - некорректный индекс, а i - корректный
  • Когда i и -i корректны и не 0 - ось симметрии проходит не по середине строки

Пример


        
s = "abcdefghij"
for i in range(len(s)):
    print(i, s[i], s[-i])

Тип значения по индексу

  • По общему смыслу одиночный символ и строка - разные объекты
  • Символ - это одиночная сущность
  • У него есть Unicode-код
  • У него может быть визуальное представление
  • Или связанное с ним действие - для управляющих символов
  • С ним сложно ассоциировать понятие "длины"
  • По символу странно итерироваться - как странно итерироваться по числу
  • Символ атомарен, его сложно разделить на составные части

Тип значения по индексу

  • А строка - набор символов
  • Он может состоять из 0, 1, 2, ... элементов
  • Для строки естественны операции итерирования или взятия элемента по индексу
  • Для строки естественно иметь длину
  • Для строки в целом странно ставить вопрос "какой у нее Unicode-код ?"
  • И поэтому естественно и логично иметь два отдельных типа данных: символ и строка
  • Примерно все языки программирования так и делают
  • Но не Python

Тип значения по индексу

  • В Python нет отдельного типа для символа
  • Результат индексирования строки - строка длиной 1
  • У результата индексирования есть длина - 1
  • По нему можно проитерироваться - это будет цикл с одной итерацией
  • Встроенная функция ord требует, чтобы аргументом была строка длиной 1

Пример


s = "abcdefghij"
print(type(s), type(s[0]))
for v in s[0]:
    print(v)
for v1 in s:
    print('v1', v1)
    for v2 in v1:
        print('v2', v2)
        for v3 in v2:
            print('v3', v3)

Вырезка

  • Может потребоваться подстрока
  • Для взятия подстроки есть операция вырезки
  • В квадратных скобках используем двоеточие
  • Или два двоеточия
  • Базовый вариант: s[i:j]
  • i - начальный индекс, j - позиция после конечного индекса
  • (Полуоткрытый интервал)

Пример


          
s = "abcdefghij" 
print(s[0:4])
print(s[3:5])

for i in range(len(s)):
    for j in range(i, len(s)):
        print(i, j, s[i:j])

Отсутствующая граница

  • Число слева или справа от : может отсутствовать
  • Если отсутствует слева - это префикс
  • Это отсутствует справа - это суффикс
  • Числа могут отсутствовать с обоих сторон
  • Это будет копия строки
  • Практического смысла в копии строки мало - строки неизменяемы
  • Но чисто технически это корректная операция

Пример



s = "abcdefghij"
print(s[:4])
print(s[3:])
print(s[:])

for i in range(len(s)):
    print('prefix to pos', i, s[:i])
    print('suffix from pos', i, s[i:])

Пустые вырезки и выходы за границы

  • (Пока только для неотрицательных индексов границ)
  • Если левая граница не меньше, чем длина строки - получим пустую строку
  • Если правая граница не больше, чем левая - получим пустую строку
  • Если правая граница больше, чем длина строки - это корректная вырезка
  • В нее входит то, что реально вошло в строку

Пример



s = "abcdefghij"
print(s[10:])
print(s[15:20])
print(s[15:10])
print(s[5:5])
print(s[7:2])

for i in range(len(s), len(s) + 5):
    print(f"{i}: <{s[3:i]}>")

Отрицательные индексы в вырезках

  • Зависит от абсолютного значения индекса
  • Если абсолютное значение НЕ превышает длины строки - реальный индекс получает как для обычного индекса
  • Если превышает - это просто отрицательный индекс, указывающий в воображаемую позицию левее начала строки
  • Для реально отрицательного индекса действуют зеркальные аналоги правил для слишком большого индекса
  • Если правая граница реально отрицательная - получим пустую строку
  • Если левая граница реально отрицательная - это корректная вырезка
  • В нее входит то, что реально вошло в строку

Пример



s = "abcdefghij"

for i in range(20, -20, -1):
    for j in range(i, -20, -1):
      print(f"i={i}, j={j}: <{s[i:j]}>")
      print(f"j={i}, i={j}: <{s[j:i]}>")
      print()

Пример



s = "abcdefghij"

for i in range(-20, 20, 1):
    print(f"i={i}, i + 1={i + 1}: <{s[i:i+1]}>")
