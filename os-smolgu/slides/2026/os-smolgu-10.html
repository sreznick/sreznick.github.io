Pipe

  • pipe - средство общения между "близкими родственниками"
  • Классическое применение - организация конвейера в bash
  • Может использоваться при передаче данных между родителем и ребенком
  • Реализуется на основе системного вызова pipe

Пример



import os

result = os.pipe()

print(result)

Что это и зачем ?

  • Сначала ответим на вопрос "что ?"
  • При успешном вызове системный вызов pipe создает и возвращает 2 файловых дескриптора
  • Файловый дескриптор - это число
  • Имеет смысл только в контексте конкретного процесса
  • Уникальным образом идентифицирует один из каналов ввода-вывода, действующих в данный момент в процессе

Файловые дескрипторы

  • Название не очень точное
  • Файловому дескриптору не всегда соответствует файл в строгом смысле слова
  • Процесс начинает работу с тремя уже готовыми дескрипторами
  • 0 - stdin
  • 1 - stdout
  • 2 - stderr

Файловые дескрипторы

  • При открытии файла создается новый дескриптор
  • При закрытии файла дескриптор освобождается
  • Освобожденный дескриптор может быть переиспользован

Файловые дескрипторы

  • Обычно при Python-разработке напрямую с дескрипторами не работают
  • Потому что работают с файловыми объектами
  • Это уже Python-надстройка
  • Но из файлового объекта можно получить его дескриптор - через метод fileno
  • И из дескриптора можно получить файловый объект: os.fdopen

Файловые дескрипторы

  • Дескрипторы - это фактически индексы в массивах в структурах ядра
  • У каждого процесса свой массив дескрипторов
  • Он имеет фиксированный размер
  • По умолчанию - 1024
  • Если открывать дескрипторы и не закрывать - можно получить отказ в очередном открытии

Пример


          
import os 

for i in range(1000):
    print('i', i)
    result = os.pipe()
    print('result', result)
    print()
          
print(result)

Утечка дескрипторов

  • Принципиально две причины
  • Первая - небрежная работа с дескрипторами (открываем и бросаем)
  • Вторая - реально нужная очень активная работа с файлами
  • Второй случай бывает не так часто
  • Можно поднять лимит для процесса
  • Но это не универсальное решение "не глядя"

На уровне выше

  • Можно попробовать открывать файлы в Python и не закрывать их
  • И делать это в цикле
  • Весьма вероятно, что исчерания дескрипторов не будет
  • При условии, что файловые объекты не сохраняются

На уровне выше

  • Это вследствие того, что освобождение дескрипторов привязано к сборке мусора
  • Когда на файловый объект не остается ссылок - происходит освобождение памяти
  • И автоматически вызывается close
  • Это решение на уровне приложения в конкретной среде
  • Это может защищать ситуативно, но на это не нужно полагаться

Пример


import os

for i in range(10000):
    print('i', i)
    result = os.pipe()
    print('result', result)
    print()
    os.close(result[0])
    os.close(result[1])

print(result)

Смысл дескриптора

  • Файловый дескриптор - идентификатор абстракции
  • Основные операции абстракции - read и write
  • За дескриптором стоит конкертная реализация абстракции
  • В случае классических файл read/write делегируются в файловую систему

Смысл дескриптора

  • pipe - файловый интерфейс к буферу памяти
  • Через write пишем, через read читаем
  • В Python есть read/write над файловыми объектами
  • С буферизацией и т.п.
  • Мы используем более низкоуровневые os.read/os.write

Пример


import os

r, w = os.pipe()

os.write(w, 'hello'.encode())
for i in range(5):
    print(os.read(r, 1))

Разберем

  • Размер канала ограничен
  • При переполнении ждем, пока данные будут прочитаны
  • В нашем случае - никогда
  • В общем случае - зависим от скорости второго процесса
  • Также возможен дедлок

Второй процесс

  • В примерах выше процесс общается сам с собой
  • Что не так интересно в плане практического применения
  • И на первый взгляд не совсем понятно, как вовлечь второй процесс
  • Создаем дескрипторы внутри одного процесса
  • А дескрипторы ограничены одним процессом

Пример


import os
import time

r, w = os.pipe()
child = os.fork()

if child > 0:
    print('parent')
    data = os.read(r, 1)
    print('read', data)
else:
    print('child')
    time.sleep(5)
    data = os.write(w, 'abc'.encode())
    print('written')

Пример


import os
import time

r, w = os.pipe()

child = os.fork()

if child > 0:
    print('parent')
    for i in range(1000):
        data = os.write(w, 'p'.encode())
    data = os.read(r, 1000)
    print('read', data)
else:
    print('child')
    for i in range(1000):
        data = os.write(w, 'c'.encode())
    print('written')
