Разделяемая память

  • Можно создать область памяти, доступную нескольким процессам
  • Для этого тоже есть свой механизм
  • И тоже в двух модификациях - POSIX и IPC
  • Часто используется в сочетании с семафорами/локами

Пример



import os
import time
import sysv_ipc
import sys

key = 123456

sm = sysv_ipc.SharedMemory(key, sysv_ipc.IPC_CREAT, size=1024)
sm.write(sys.argv[1].encode())

Пример



import os
import time
import sysv_ipc

key = 123456

sm = sysv_ipc.SharedMemory(key)
print(sm.read(10))

Очереди

  • Имея локи и разделяемую память - можно создавать разннобразные структуры для передачи данных
  • Классическая структура - очередь
  • Ей соответствует шаблон проектирования producer-consumer
  • Linux предлагает два варианта готовых очередей: POSIX и SysV

Сети в контексте ОС

  • Сетевые возможности в контексте ОС можно понимать как еще один вариант межпроцессоного взаимодействия
  • Только процессы могут жить на разных узлах
  • Архитектура предполагает разнообразие протоколов на разных уровнях
  • На практике некоторые протоколы доминируют
  • И в силу этого особо важны

Общие принципы

  • Коммутация пакетов (не каналов)
  • Уровни протоколов
  • Нижний уровень - "физический"
  • Доступ к сетевой среде
  • Витая пара, коаксиальный кабель, WiFi

Общие принципы

  • Уровень выше - локальные сети
  • Пример: Ethernet
  • Адресация - MAC-адрес
  • Разделяемая физическая среда
  • Трафик между узлами физически доступен третьм узлам в той же сети

Общие принципы

  • Уровень выше - передача данных между локальными сетями
  • IP-адреса
  • Маршрутизация
  • Проблемы доставки: потери пакетов, пропажа пакетов, дублирование, наружение порядка

IP и UDP

  • IP - глобальный транспортный протокол
  • Старается обеспечить доставку пакетов от узла к узлу - через промежуточные сети
  • Адресация - от узла к узлу
  • Не учитывает приложения
  • UDP - легкая надстройка над IP
  • Появляется понятие "port" - для адресации приложения внутри хоста

TCP

  • TCP - дорогая надстройка над IP
  • Обеспечивает поток данных: файло-подобный интерфейс к сети
  • За кулисами - нумерация пакетов, перепосылки при необходимости
  • Основа клиент-серверной архитектуры
  • ВАЖНО: гарантии TCP работают в пределах соединения и в рамках сеанса

Реализация в ОС

  • IP и ниже - реализуется в ядре Linux
  • TCP и UDP - обычно управляется приложением
  • Выше TCP и UDP - прикладные протоколы
  • Для приложения используется абстракция сокета

Реализация в ОС

  • Сокет бывает клиентским и серверным
  • Сокет привязан к точке доступа
  • Точка доступа - в первую очередь сетевой интерфейс
  • Или что-то синтетически-виртуальное

Серверный сокет

  • Жизненный цикл: создание, bind, listen, accept
  • После accept получаем клиентский сокет у себя
  • Соединенный с удаленным клиентским сокетом
  • И возможность вернуть в accept
  • Вместе с необходимостью общаться по клиентскому сокету

Пример


import socket

server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server_socket.bind(('localhost', 65432))
server_socket.listen(5)
print("Server is listening on port 65432...")

Пример: datagram server



sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
server_address = ('localhost', 10000)
sock.bind(server_address)

print(f"Server started on {server_address[0]} port {server_address[1]}")

while True:
    data, address = sock.recvfrom(4096)

    print(f"Received {len(data)} bytes from {address}")

    if data:
        sent = sock.sendto(data, address)

Пример: datagram server



import socket

server_address = ('127.0.0.1', 10000)
message = b"Hello, Server!"

with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as client_socket:
    try:
        print(f"Sending: {message}")
        client_socket.sendto(message, server_address)

        data, server = client_socket.recvfrom(1024)
        print(f"Received: {data.decode()} from {server}")

    except Exception as e:
        print(f"Error: {e}")

Сетевые утилиты

  • Есть ряд утилит для работы с сетью
  • Или для более широких целей, включая работу с сетью
  • Во многих требуется настройка в зависимости от протокола - IPv4 или IPv6
  • Через опцию или через IPv6-версию утилиты

ping

  • Реализация протокола ICMP
  • Организует передачу посылку пакетов до узла и обрабтно
  • Показывает принципиальную доступность узла
  • И качество соединения
  • Заодно помогает узнать IP адрес по имени хоста

ip

  • Показывает настройки на IP-уровне
  • Может их менять (нужна осторожность)
  • Пример: ip addr
  • Показывает данные о досутпных сетевых подключениях

nc

  • nc - netcat
  • Создает простых клиентов и простые серверы
  • Пример сервера: nc -l 1234
  • Пример клиента: nc -N 127.0.0.1 1234
  • UDP-клиент: nc -u localhost 10000

netstat

  • Инофрмацмя о настройках и статистике
  • Таблица маршрутизации: netstat -r
  • Сетевые интерфейсы: netstat -i
  • Статистика: netstat -s
  • Слушающие сокеты: netstat -l

lsof

  • Показывает открытые файловые дескрипторы по процессам
  • Пример: lsof -iUDP
  • Пример: lsof -iTCP -sTCP:LISTEN
  • lsof /dev/root

curl, wget

  • Исполняяют HTTP-запросы
  • HTTP протокол поверх TCP
  • Основа работы браузеров
  • И микросервисной архитектуры
  • curl больше подходит для текстовых ответов
  • wget больше подходит для скачивания файлов

Параллелизм

  • Острее всего на стороне сервера
  • Для клиента тоже бывает актуально
  • Лобовое решение - fork
  • Дорогое и плохо масштабируемое
  • Можно лечить пулингом
  • Но есть неудобства с разными адресными пространствами

Параллелизм

  • Альтернатива - нити
  • В терминах Linux это системный вызов clone
  • В библиотеках это прорастает в виде концепции Thread
  • Воспринимается как дихотомия нить/процесс
  • На уровне ОС вариантов больше, чем 2

Параллелизм

  • В Linux нити по стоимости сопоставимы с процессами
  • Но их легче пулить
  • Оборотная сторона: нить сложнее аккуратно завершать
  • На нити нет аналога kill -9 - по принципиальным причинам

Параллелизм

  • Еще одна альтернатива - асинхронная работа
  • Базируется на системном вызове select
  • В цикле наблюдаем за изменением состоянием дескрипторов
  • Обрабатываем тех, у кого изменилось состояние
  • Нужна хороша языково-библиотечная поддержка
