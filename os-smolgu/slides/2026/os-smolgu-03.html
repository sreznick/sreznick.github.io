stdout/stderr

  • Многие приложения что-то порождают в ходе работы
  • Приложение может открыть файл и порождать его содержимое
  • Приложение может что-то "просто печатать": print("hello") в Python
  • В Unix-системах для приложения запись в файл и "просто печать" не очень отличимы
  • При записи в файл приложение должно открыть файл и получить файловый дескриптор (число)
  • И использовать его как параметр системного вызова write
  • При "просто печати" используется предоставленный ОС файловый дескриптор - это называется "стандартный выход"
  • В базовом сценарии то, что идет на стандартный выход, печатается на терминале

stdout/stderr

  • Базовый сценарий можно изменить, не затрагивая приложение
  • Например, направить стандартный выход в файл: cat file.txt > file2.txt
  • Это универсальный механизм для всего, что запускается из командной строки
  • Если файл существовал - старое содержимое будет потеряно
  • cat file.txt > file.txt скорее всего приведет к пустому файлу
  • Можно перенаправить стандартный выход в конец файла, если он существовал: cat file.txt >> file2.txt
  • cat file.txt >> file.txt скорее всего приведет к бесконечному росту файла

stdout/stderr

  • Попробуем получить содержимое несуществующего файла: cat not-existing.txt > copy.txt
  • Увидим на терминале сообщение о ошибке
  • Оно никуда не перенаправится
  • Это так и должно быть
  • ОС дает приложению два разных дескриптора для стандартного выхода
    • 1 - stdout
    • 2 - stderr

stdout/stderr

  • Можно перенаправить stderr: cat not-existing.txt 2> copy.err
  • Можно перенаправить оба: cat not-existing.txt > copy.txt 2> copy.err
  • Оба перенаправляем в один файл: cat file.txt >result 2>&1
  • А это немного другое: cat file.txt 2>&1 >result
  • На stdout обычно пишется то, что порождено штатной логикой приложения
  • На stderr - сопутствующая информация, не обязательно ошибки
  • Часто на stderr пишется прогресс приложения

stdin

  • С чтением ситуация похожая
  • Приложение может читать данные из конкретного файла
  • А может - со стандартного входа (input() в python)
  • На входе нет аналога потока для ошибок
  • Номер дескриптора стандартного входа - 0

stdin

  • Перенаправление стандартного входа - через <
  • Многие утилиты, предполагающие входные данные, используют смешанную схему
  • Если в аргументах командной строки указаны файлы - используются файлы
  • Если файлов нет - используется стандартный вход
  • Иногда - вместо имени файла обозначает использование стандартного входа

Конвейер

  • Есть вариант параллельного запуска двух приложений
  • Стандартный выход одного приложения направить на стандартный вход другого
  • Пример (оценили число файлов в текущем каталоге): ls | wc -l
  • Можно понимать как алгебру - формальную систему с выражениями, которые строятся по правилам
  • | - одна из операций
  • Но не единственная

Другие операции

  • ; - последовательное исполнение
  • Пример: echo 123 ; cat file.txt
  • && - логическое И
  • || логическое ИЛИ
  • Последние две предполагают логическую интерпретацию исполнения команды
  • И оно есть

Логическая интерпретация

  • Каждый процесс завершается с числовым статусом
  • Это факт уровня операционной системы, не оболочки
  • Оболочки его используют
  • Статус определяется разработчиком приложения
  • Если это скрипт на интерпретируемом языке - как разработчиком интерпретатора, так и разработчиком скрипта
  • Подразумевается, что статус 0 - успешное завершение
  • Статус не 0 - неудачное завершение

Логическая интерпретация

  • Для C/С++-приложения статус определяется возвращаемым значением функции main
  • Или параметром, переданным в exit
  • Если main ничего не возвращает и exit не вызывается - статус будет равен 0
  • Для Python-скрипта, завершающегося во sys.exit статус определяется параметром
  • Обычное завершение - статус 0

Логическая интерпретация

  • Операнды логических операций исполняются последовательно
  • Второй операнд не исполняется если результат понятен без него
  • Статус 0 - это true, статус не 0 - это false
  • Частый прием: echo "going to do something" ; ( do-something && echo success || echo fail)

Работа с текстом

  • cat - выводит на стандартный выход содержимое файлов как есть
  • tac - выводит "записи" файла в обратноом порядке
  • По записями по умолчанию понимаются строки
  • nl - нумерация строк
  • Пример совместного использования через конвейер: tac | nl | tac
  • od - дамп бинарных файлов
  • Пример применения: od -A x -t x1z -v /bin/cat

Работа с текстом

  • head - выводит на стандартный выход первые n строк/символов
  • tail - выводит на стандартный выход последние n строк/символов
  • split - разбивает файл на куски по числу байтов/строк
  • wc - выводит на стандартный выход количество байт/строк/слов
  • csum, cksum, md5sum, b2sum, sha1sum, sha2 - выводят на стандартный выход разные вариаиты контрольных сумм
  • sort - выводит на стандартный выход отсортированные строки
  • shuff - выводит на стандартный выход перемешанные строки
  • uniq - убирает повторяющиеся строки

Работа с текстом

  • cut - выводит на стандартный выход заданные фрагменты строк файла
  • Пример: cut -d : -f 1,5 /etc/passwd
  • paste - склеивает соответствующие строки разных файлов или последовательные строки одного
  • Пример: paste f1.txt f2.txt
  • Пример: cut -d : -f 1,5 /etc/passwd | paste - -
  • join - соединяет строки разных файлов по общему столбцу

Работа с текстом

  • tr - замена символа на символ или удаление символов
  • Пример: tr -cs "[:alpha:]" "\n"
  • Пример: tr "[:lower:]" "[:upper:]"
  • Пример: tr -cd "[:print:]"
  • expand - замена табуляций на пробелы
  • unexpand - пробелов на табуляции

Работа с текстом

  • echo - уже знаем
  • Существует как исполняемый файл независимо от bash
  • printf - выводит на стандартный выход форматную строку
  • yes - бесконечно выводит на стандартный выход y или yes
  • tee - сохраняет входной текст в файл и воспроизводит его на стандартный выход
  • seq - выводит на стандартный выход последовательность чисел

grep/sed/awk

  • grep: Globally search for a Regular Expression and Print matching lines
  • Группа утилит для поиска строк по образцу
  • sed: Stream EDitor
  • Утилиты для выборочного изменения/удаления строк
  • awk: Aho, Weinberger, Kernighan
  • Названа по именам создателей
  • Развивает идеи sed
  • Плавно переходя в язык программирования

grep

  • grep - группа из 4 утилит
  • fgrep - поиск подстроки, без регулярных выражений
  • rgrep - можно передать каталог вместо файла
  • И будет поиск по всем файлам из подкаталогов
  • egrep - используется более развитая форма регулярного выражения
  • Более фориально
    • fgrep - это grep -F
    • rgrep - это grep -r
    • egrep - это grep -E

grep

  • Регулярные выражения - способ задания языков (подмножест универсального множества над данным алфавитом)
